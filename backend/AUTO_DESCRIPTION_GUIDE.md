# Автоматична генерація описів товарів

## Огляд

Система автоматичної генерації описів дозволяє створювати професійні описи товарів на основі їх назви та категорії. Підтримується два режими:

1. **AI генерація** (через OpenAI GPT-4) - створює унікальні, детальні описи
2. **Шаблонна генерація** - створює базові описи за шаблоном

---

## Налаштування

### 1. API ключ OpenAI (для AI генерації)

Для використання AI генерації потрібно отримати API ключ від OpenAI:

1. Зареєструйтеся на https://platform.openai.com/
2. Створіть API ключ в розділі API Keys
3. Додайте ключ до `.env` файлу або в `docker-compose.yml`:

**Варіант 1: Через .env файл**
```env
OPENAI_API_KEY=sk-your-api-key-here
```

**Варіант 2: Через docker-compose.yml**
```yaml
backend:
  environment:
    - OPENAI_API_KEY=sk-your-api-key-here
```

> ⚠️ **Важливо**: Якщо API ключ не налаштовано, система автоматично використовує шаблонну генерацію

---

## Використання в адмін-панелі Django

### Автоматична генерація при створенні

При створенні нового товару через адмін-панель:

1. Заповніть **назву товару** та **категорію**
2. Залиште поля **"Опис"** та **"Короткий опис"** порожніми
3. Збережіть товар
4. Система автоматично згенерує обидва описи

### Масова генерація для існуючих товарів

1. Перейдіть до списку товарів в адміністраторській панелі
2. Виберіть товари (checkboxes)
3. В випадаючому меню "Дії" оберіть:
   - **🤖 Згенерувати описи (AI)** - для AI генерації
   - **📝 Згенерувати описи (Шаблон)** - для шаблонної генерації
4. Натисніть "Виконати"

## Використання через API

### Створення товару з автогенерацією

**Endpoint**: `POST /api/products/`

**Приклад запиту**:
```json
{
  "name": "Яблуня Голден Делішес",
  "category": 1,
  "price": 150.00,
  "stock": 50,
  "auto_generate_description": true
}
```

**Параметри**:
- `auto_generate_description` (boolean, optional) - чи генерувати опис автоматично (за замовчуванням: `true`)
- `description` (string, optional) - якщо вказано, автогенерація не спрацює
- `short_description` (string, optional) - якщо вказано, автогенерація не спрацює

### Приклад повної відповіді

```json
{
  "id": 123,
  "name": "Яблуня Голден Делішес",
  "slug": "yablunia-golden-delishes",
  "category": 1,
  "description": "Яблуня Голден Делішес з категорії Плодові дерева - якісний саджанець для вашого саду...",
  "short_description": "Яблуня Голден Делішес - популярний сорт з золотистими плодами...",
  "price": "150.00",
  "stock": 50,
  "sku": "SKU-A5B9C2D1"
}
```

## Як це працює

### AI генерація (OpenAI GPT-4)

1. Система формує промпт з назвою товару та категорією
2. Відправляє запит до OpenAI API
3. Отримує професійний опис українською мовою
4. Парсить відповідь на повний та короткий опис
5. Зберігає в базу даних

**Приклад згенерованого опису**:

```
Яблуня Голден Делішес - один з найпопулярніших десертних сортів яблук у світі. 
Цей саджанець відзначається високою врожайністю та відмінними смаковими якостями плодів.

Плоди середніх розмірів, золотисто-жовтого кольору з характерною оржавленістю біля 
плодоніжки. М'якоть соковита, солодка з тонким ароматом. Вага плоду 140-180 грам.

Дерево середньорослих розмірів з широкопірамідальною кроною. Зимостійкість висока, 
витримує морози до -25°C. Стійкість до парші та борошнистої роси середня.

Врожайність: 80-120 кг з дерева. Плодоносити починає на 3-4 рік після посадки. 
Плоди дозрівають у вересні та зберігаються до березня.

Рекомендації по догляду: регулярний полив, підживлення органічними добривами, 
санітарна обрізка навесні. Вимагає сонячного місця посадки та родючих ґрунтів.
```

### Шаблонна генерація

Якщо OpenAI API недоступний або вимкнено, використовується базовий шаблон з загальною інформацією про саджанці.

## Обмеження та рекомендації

### Обмеження OpenAI API

- **Вартість**: Кожен запит споживає токени (приблизно $0.001-0.005 за опис)
- **Rate limits**: OpenAI має ліміти на кількість запитів
- **Латентність**: Генерація займає 2-5 секунд

### Рекомендації

1. ✅ Використовуйте AI генерацію для ключових товарів
2. ✅ Використовуйте шаблонну генерацію для масових товарів
3. ✅ Завжди перевіряйте та редагуйте згенеровані описи
4. ✅ Додавайте специфічні характеристики вручну після генерації
5. ❌ Не використовуйте AI генерацію для тисяч товарів одночасно

## Логування та налагодження

Логи генерації описів можна переглянути:

```bash
# Логи backend контейнера
docker-compose logs -f backend

# Фільтрація логів генерації
docker-compose logs backend | grep "генеровано"
```

**Типові повідомлення логів**:
- `Згенеровано AI опис для товару: [назва]` - успішна AI генерація
- `OPENAI_API_KEY не знайдено, використовується шаблонний опис` - fallback на шаблон
- `Помилка при генерації AI опису: [error]` - помилка API

## FAQ

**Q: Чи можна використовувати без OpenAI?**  
A: Так, система автоматично перемикається на шаблонну генерацію

**Q: Скільки коштує використання OpenAI?**  
A: Приблизно $0.001-0.005 за один опис (залежить від довжини)

**Q: Чи можна змінити мову генерації?**  
A: Так, відредагуйте промпт в файлі `products/utils.py`

**Q: Описи генеруються для всіх полів?**  
A: Генеруються `description` та `short_description`. Інші поля заповнюються вручну

**Q: Що робити якщо опис не підходить?**  
A: Просто відредагуйте його вручну в адмін-панелі

## Технічні деталі

### Файли проекту

- `products/utils.py` - логіка генерації описів
- `products/serializers.py` - ProductCreateSerializer
- `products/admin.py` - admin actions для масової генерації
- `products/views.py` - API endpoints

### Використовувана модель AI

- **Модель**: GPT-4o-mini (більш доступна та швидка версія)
- **Max tokens**: 600
- **Temperature**: 0.7 (баланс між креативністю та точністю)

### Безпека

- API ключ зберігається в змінних оточення
- Ніколи не комітьте `.env` файл з ключем
- У production використовуйте захищені secrets
