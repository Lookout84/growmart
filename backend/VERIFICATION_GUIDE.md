# Функціональність верифікації користувачів

## Описання

Функціональність верифікації користувачів дозволяє адміністраторам перевірити та активувати облікові записи зареєстрованих користувачів через адмін панель Django.

## Функції

### 1. **Індивідуальна верифікація**
- Адміністратор може верифікувати одного користувача, позначивши чекбокс `is_verified` у формі редагування користувача

### 2. **Масова верифікація**
- Адміністратор може вибрати кількох користувачів зі списку та застосувати дії:
  - **✓ Верифікувати вибраних користувачів** - верифікує всіх вибраних користувачів
  - **✗ Скасувати верифікацію вибраних користувачів** - скасовує верифікацію для вибраних користувачів

### 3. **Статус верифікації**
- У списку користувачів показується статус верифікації з емодзі:
  - ✅ Верифіковано
  - ❌ Не верифіковано

### 4. **Фільтрація**
- Адміністратор може фільтрувати користувачів за статусом верифікації
- Фільтри: `is_verified`, `is_active`, `is_staff`, `is_superuser`

### 5. **Вдосконалений інтерфейс**
- Поле `is_verified` розташоване у розділі "Верифікація та контакти"
- Описання поля пояснює, що верифікація дозволяє користувачеві доступ до всіх функцій
- Розділ адреси згорнутий за замовчуванням (`collapse`)

### 6. **Автоматичні емейли**
- Коли користувача верифікують через адмін панель, йому автоматично відправляється емейл з повідомленням про успішну верифікацію
- Емейл містить:
  - Привітання
  - Інформацію про успішну верифікацію
  - Список доступних функцій
  - Посилання на сайт

## Як користуватися

### Верифікація одного користувача:
1. Перейдіть на адмін панель: `http://localhost:8000/admin/`
2. Перейдіть до розділу "Користувачі"
3. Натисніть на користувача, якого хочете верифікувати
4. У розділі "Верифікація та контакти" позначте чекбокс "is_verified"
5. Натисніть "Зберегти"

### Масова верифікація:
1. Перейдіть на адмін панель: `http://localhost:8000/admin/`
2. Перейдіть до розділу "Користувачі"
3. Виберіть користувачів, позначивши чекбокси в таблиці
4. У меню "Дії" виберіть:
   - "✓ Верифікувати вибраних користувачів" для верифікації
   - "✗ Скасувати верифікацію вибраних користувачів" для скасування верифікації
5. Натисніть кнопку "Перейти"

## Деталі реалізації

### Файли змінені/створені:
- `backend/users/admin.py` - Оновлений адмін интерфейс з діями масової верифікації
- `backend/users/signals.py` - Сигнали для відправки емейлів при верифікації (новий)
- `backend/users/apps.py` - Реєстрація сигналів (оновлено)
- `backend/users/templates/emails/verification_confirmation.html` - Шаблон емейлу (новий)

### Модель User:
- Поле `is_verified` вже присутнє в моделі User
- Тип: BooleanField
- За замовчуванням: False

### API:
- Поле `is_verified` включене в UserSerializer як read-only для звичайних користувачів
- Адміністратори можуть змінювати цей статус через адмін панель

## Налаштування емейлів

Для відправки емейлів переконайтеся, що у `settings.py` налаштовані параметри:
```python
EMAIL_BACKEND = 'django.core.mail.backends.console.EmailBackend'  # для розробки
# або
EMAIL_BACKEND = 'django.core.mail.backends.smtp.EmailBackend'  # для продакшену
EMAIL_HOST = 'smtp.gmail.com'
EMAIL_PORT = 587
EMAIL_USE_TLS = True
EMAIL_HOST_USER = 'your-email@gmail.com'
EMAIL_HOST_PASSWORD = 'your-app-password'
DEFAULT_FROM_EMAIL = 'noreply@zelenykutochok.ua'
```

## Примітки

- Верифікація необхідна для повного функціонування платформи
- При верифікації користувача автоматично відправляється емейл (якщо налаштований email backend)
- Адміністратори можуть в будь-який момент скасувати верифікацію користувача
